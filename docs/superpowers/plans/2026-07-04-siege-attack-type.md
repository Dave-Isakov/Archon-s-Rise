# Siege Attack Type Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a second, wound-free attack type ("Siege") that spends its own scarce stat pool and cannot be improvised, giving the player a real Normal-vs-Siege choice against every enemy.

**Architecture:** The risky decision logic (does the attack defeat the enemy, and how many wounds does it inflict) is extracted into a pure, Unity-independent `CombatRules` class so it is fully unit-testable. `Siege` becomes a first-class `StatType` with its own `playerSiege` accumulator that flows through the existing `int[]` stat pipeline (`GetCardStats`/`GetUnitStats` → `AssignPlayerStats`). The combat UI and event wiring mirror the existing Fight path.

**Tech Stack:** Unity 6000.5.1f1, C#, NUnit (EditMode), assembly-definition-scoped pure logic under `ArchonsRise.CardPlay` / `ArchonsRise.Enums`.

## Global Constraints

- **Siege spends the Siege pool only; Normal spends the Attack pool only.** No Attack↔Siege conversion.
- **A Siege kill is ALWAYS wound-free**, regardless of Defend vs enemy Attack.
- **A Siege kill grants identical rewards to a Normal kill** — no reward/XP/crystal/influence penalty.
- **Siege is never improvisable.** The Improvise panel offers only Attack/Defend/Influence/Explore; do not add Siege to it. No enforcement code is needed — leave `ImprovisePanel` untouched.
- **Stat array order is positional:** `[0]=Attack, [1]=Defend, [2]=Influence, [3]=Explore, [4]=Siege`. Every producer and consumer of the stat array must agree on this order and a length of **5**.
- **One-exchange combat is preserved:** no persistent enemy HP, no multi-turn chipping, no Doom-clock hook.
- **Never hand-edit scene/prefab/asset YAML.** Editor wiring (buttons, HUD labels, ScriptableObject event assets, listener responses) is handed to the user as step-by-step instructions (Task 6). `.meta` files for new scripts are generated by Unity on editor focus — do not author them by hand.

### Verification model (this project)

The Unity editor is usually open, holding the project lock, so `Unity.exe -runTests -batchmode` fails. Therefore:
- **Pure logic** (`CombatRules`, `StatPalette`) is RED/GREEN-verified from the CLI with the **Mono `mcs` harness** (Task 1 gives exact commands).
- **Unity-coupled changes** (ScriptableObject arrays, the `Player` MonoBehaviour, UI, event wiring) are verified by **successful compilation** plus the **user confirming in Window ▸ General ▸ Test Runner / in-play behavior**. These tasks note it explicitly.

```csharp
// Pure combat resolution rules. No scene/Unity dependency so the wound-free
// Siege guarantee is unit-testable via the CLI pure-test harness.
public enum AttackKind { Normal, Siege }

public static class CombatRules
{
    // Normal spends the Attack pool; Siege spends the Siege pool. Either defeats
    // the enemy when its own pool covers the enemy's HP. Pools never cross over.
    public static bool CanDefeat(AttackKind kind, int attackPool, int siegePool, int enemyHP)
        => kind == AttackKind.Siege ? siegePool >= enemyHP : attackPool >= enemyHP;

    // The counterattack wound the player takes on a defeat. Siege is always
    // wound-free. Normal wounds when Defend falls short of the enemy's Attack,
    // one wound per HP-sized bite of the shortfall (matches the original loop).
    public static int WoundCount(AttackKind kind, int defend, int enemyAttack, int playerHP)
    {
        if (kind == AttackKind.Siege) return 0;
        if (defend >= enemyAttack) return 0;
        int wounds = 0;
        for (int i = 0; i < enemyAttack - defend; i += playerHP) wounds++;
        return wounds;
    }
}
```

- [ ] **Step 1: Replace the resolution methods**

In `Assets/Scripts/GameObjectScripts/PlayerScripts/Player.cs`, replace `ValidatePlayerAttackToEnemyHP` (~line 229) and `CheckWounds` (~line 323) with a shared resolver plus two entry points. Delete the old `CheckWounds` method (its logic now lives in `CombatRules.WoundCount`; the only live caller was `ValidatePlayerAttackToEnemyHP`):

```csharp
    // Normal attack: spends the Attack pool, and the enemy's counterattack can wound
    // the player when Defend falls short. Entry point wired to the Fight button.
    public void ValidatePlayerAttackToEnemyHP(EnemyCard enemy)
    {
        ResolveAttack(enemy, AttackKind.Normal);
    }

    // Siege attack: spends the Siege pool and is always wound-free. Entry point
    // wired to the Siege button.
    public void SiegeEnemy(EnemyCard enemy)
    {
        ResolveAttack(enemy, AttackKind.Siege);
    }

    void ResolveAttack(EnemyCard enemy, AttackKind kind)
    {
        int hp = enemy.enemySO.enemyHP;
        if (!CombatRules.CanDefeat(kind, playerAttack, playerSiege, hp))
        {
            string stat = kind == AttackKind.Siege ? "Siege" : "Attack";
            GameManager.Instance.ValidationMessage($"You need {hp} {stat} in order to defeat this monster.");
            return;
        }

        if (kind == AttackKind.Siege) playerSiege -= hp;
        else                          playerAttack -= hp;

        int wounds = CombatRules.WoundCount(kind, playerDefend, enemy.enemySO.enemyAttack, playerHP);
        for (int i = 0; i < wounds; i++)
            onDefeat_WoundPlayer.Raise(enemy);

        // Only a Normal attack takes the counterattack against Defend.
        if (kind == AttackKind.Normal) playerDefend -= enemy.enemySO.enemyAttack;

        if (wounds > 0)
            GameManager.Instance.ValidationMessage($"{enemy.enemySO.name} has been destroyed! You are wounded {wounds} times!");

        OnEnemyDefeat_GetRewards.Raise(enemy);
    }
```
